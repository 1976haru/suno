# v2 → v3 마이그레이션 노트

이 문서는 Suno Weaver Studio v2에서 v3로 넘어오면서 무엇이 바뀌었는지, 그리고 사전 검증 리포트(작업 지시문 0장)에 있던 결함들이 실제로 어떻게 처리됐는지 정리한 것입니다.

## v3.1 업데이트 (가사 품질 후속 작업)

v3의 39개 테스트는 전부 통과했지만, **실제로 생성된 가사를 읽어보니 문법이 깨져 있었습니다** (테스트가 "곡들끼리 다른가"만 측정하고 "가사 한 줄이 말이 되는가"는 측정하지 못했기 때문). v3.1은 이 문제를 고쳤습니다.

- **모티프(반복 이미지) 과다 사용**: 곡 하나에 `evening train` 같은 모티프가 최대 6번 등장하던 문제 → 예산제(최대 3회, 후렴 반복 포함)로 제한. 모티프가 필요 없는 슬롯은 매번 다른 일반 명사("morning", "아침", "朝" 등)로 대체.
- **영어 관사 누락**: `like evening train` (비문) → `like an evening train`으로 자동 교정하는 `likeMotif()` 헬퍼 추가.
- **후렴에 소문자 제목이 통째로 삽입되던 문제**: `one more time, christmas eve window light` → 제목에서 짧은 후크(사물 단어 1~2개)만 추출해 사용하도록 변경. `song.hookPhrase`가 실제 후렴 첫 행과 항상 일치하도록 보장.
- **제목 내부 단어 중복**: `Old Radio Morning Radio` 같은 제목 → 조합 시 단어 겹침 검사 추가.
- **[v3.1 실사용 테스트로 발견] 한국어/일본어 가사에 영어가 섞여 나오던 문제**: `situation`(상황)과 `motif`(모티프) 데이터가 영어 전용이라 한국어/일본어 곡에도 `watching the first lights come on 어딘가에서` 같은 영어 문장이 그대로 삽입되고 있었습니다. 스펙에는 없던 문제였지만 한국어/일본어 샘플을 실제로 생성해 읽어보고 발견했습니다. `{english, korean, japanese}` 3개 국어 번역 데이터로 전면 교체.
- **[v3.1 실사용 테스트로 발견] 한국어 조사 불일치**: 모티프 명사가 바뀌는데 `은/는`, `이/가`, `을/를`, `와/과` 조사가 하드코딩되어 있던 문제 → 받침 유무를 판별하는 `koParticle()` 헬퍼로 교체.
- **[v3.1 실사용 테스트로 발견] 제목의 이중 소유격**: "오래된 아침의 정적의 달력"(늙은 아침**의** 정적**의** 달력) 같은 어색한 이중 소유격 제목이 나오던 문제 → 시간어 자체에 조사가 포함되지 않도록 데이터 수정. 이 문제는 어떤 자동 테스트로도 걸러지지 않았고, `scripts/sample.ts`로 실제 30곡을 생성해 눈으로 읽어보고서야 발견했습니다.
- **단일 트랙 재생성**: 평가 에이전트가 30곡 중 일부만 반려해도 팩 전체를 다시 생성해야 했던 문제 → `regenerateTrack()`이 해당 곡만 재생성(품질/중복/유사도 자동 재검사, 최대 2회 재시도, 실패해도 최선의 결과 반환), UI에 되돌리기(undo) 1회 제공.
- **로컬 모드 안내 배너**: 생성 단계에서 로컬 템플릿 모드일 때 "가사는 조합형이라 다소 단조로울 수 있다"는 안내와 설정 바로가기 표시.
- `tests/lyricEngine.test.ts`에 19개, `tests/regenerate.test.ts`에 5개 회귀 테스트 추가 (총 64개, 기존 39개는 수정 없이 그대로 통과).
- `scripts/sample.ts`: `npx tsx scripts/sample.ts --count 30 --lang korean`처럼 실제 생성 결과 전문을 눈으로 읽어볼 수 있는 도구. 앞으로 가사 관련 변경 시 계속 사용할 것.

## 한눈에 보는 변화

- **UI**가 평면 나열식 단일 화면에서 **4단계 위저드**(채널 → 컨셉 → 생성 → 결과)로 바뀌었습니다. 사이드바에서 채널과 저장된 팩을 언제든 오갈 수 있습니다.
- **곡 수**가 10~20곡 고정에서 **1~30곡 자유 선택**으로 바뀌었습니다.
- 30곡을 생성해도 **가사와 제목이 반복되지 않습니다.** 이전 버전은 곡마다 5개 변수만 바뀌는 고정 템플릿이라 사실상 같은 가사였습니다.
- **머니코드**가 5종에서 8종으로 늘었고, 커스텀 코드 진행을 직접 입력할 수 있습니다.
- 생성 결과를 **앱 안에 저장하고 나중에 다시 불러올 수 있습니다** (IndexedDB, 자동 임시저장 포함).
- **AI 평가 에이전트**가 추가되어, Claude/ChatGPT API 연결 시 각 곡을 채점하고 재생성을 제안합니다.
- **설정 화면**에서 AI 제공자, 모델, API 키(서버/로컬 중 선택), 배치 크기를 관리합니다.
- 로컬 개발(`npm run dev`)만으로도 OpenAI/Claude 모드를 테스트할 수 있습니다.
- `vitest` 기반 자동화 테스트와 스트레스 테스트가 추가됐습니다.

### 실사용 중 추가로 발견/수정한 항목 (스펙에는 없었지만 실제 테스트로 드러남)

- **429 재시도 없음**: `openai.ts`/`anthropic.ts`가 레이트리밋 시 즉시 실패했음 → `src/providers/proxyFetch.ts`에 지수 백오프 재시도(최대 3회) 추가.
- **에러 메시지에 원본 JSON이 그대로 노출**: 클라이언트가 서버 에러 응답의 raw text를 그대로 에러 메시지에 붙였음 → 깨끗한 한국어 메시지만 추출하도록 통일.
- **배치 중 실패 시 이전 배치 결과 유실**: 3번째 배치가 실패하면 1·2번째 배치에서 만든 곡까지 전부 사라졌음 → 진행률 콜백에 부분 결과를 유지하고 결과 화면에서 확인 가능하도록 수정 (완전한 "3번째 배치만 재시도" 버튼은 후속 과제로 남음, 아래 참고).

## 알려진 한계 / 후속 과제

- **S13 배치 부분 실패 시 "3번째 배치만 재생성" 버튼**은 구현하지 않았습니다. 현재는 실패한 배치까지의 결과가 화면에 남지만, 다시 생성하면 처음부터 다시 만들어집니다.
- **S8 (IndexedDB 100팩 벌크 저장) 스트레스 테스트**는 자동화하지 않았습니다. Node.js에는 IndexedDB 구현이 없고, 이 하나의 시나리오만을 위해 `fake-indexeddb` 같은 의존성을 추가하는 것은 "의존성 최소화" 원칙과 맞지 않는다고 판단했습니다. 단일 팩의 저장/불러오기는 Playwright로 실제 브라우저에서 확인했습니다.
- **Step2Concept의 머니코드 카드**는 명시된 UX(카드 그리드 + 설명 + 미리보기)를 구현했지만, 100% 아이콘/시각 디자인까지는 다듬지 않았습니다.
